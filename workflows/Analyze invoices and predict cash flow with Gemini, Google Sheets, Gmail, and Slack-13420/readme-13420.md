## 1. Workflow Overview

**Purpose:** This workflow captures invoice data from a user-facing n8n Form, analyzes the invoice with **Google Gemini** (via LangChain nodes), performs a **fraud risk decision**, logs invoice/analysis results into **Google Sheets**, runs a second Gemini-based **cash-flow prediction**, then distributes outputs via **Slack alerting** (for risky invoices) and **Gmail reporting** (financial report delivery).

**Target use cases:**
- Accounts payable / finance ops: automate invoice intake, validation signals, and reporting.
- SMB bookkeeping: quick AI-assisted extraction and risk heuristics.
- Cash planning: predict near-term cash impact from captured invoices.

### Logical Blocks
1. **1.1 Input Reception (Invoice capture)**
   - Receives invoice input via an n8n Form Trigger.
2. **1.2 Invoice Pre-processing**
   - Normalizes/formats incoming form submission into a structured payload for AI analysis.
3. **1.3 AI Invoice Analysis (Gemini)**
   - Calls Gemini through a LangChain “Chain LLM” to extract/assess invoice details.
4. **1.4 Parsing + Risk Decision**
   - Parses the AI response into structured fields; evaluates fraud risk using an IF node.
5. **1.5 Persistence + Cash Flow Prediction**
   - Stores results in Google Sheets; runs a second Gemini chain to predict cash-flow impact.
6. **1.6 Notifications & Reporting**
   - Sends Slack alerts for risky invoices; emails a financial report via Gmail.

### 2.4 Parsing + Risk Decision

**Overview:** Converts the LLM output into deterministic structured fields and branches logic based on suspected fraud risk.

**Nodes involved:**
- **Parse AI Response**
- **Fraud Risk Check**
- **Send Alert** (risky branch)

#### Node: Parse AI Response
- **Type / role:** `n8n-nodes-base.code` — parses/validates the Gemini response.
- **Configuration (interpreted):**
  - Actual code not provided; you must implement logic such as:
    - Extract JSON from text (e.g., handle fenced code blocks).

- **Inputs / outputs:**
  - **Input:** from **AI Invoice Analyzer**
  - **Output:** Main output → **Fraud Risk Check**
- **Edge cases / failures:**
  - JSON parse errors if AI returns plain text.
  - Missing keys causing downstream IF conditions to fail.
  - Locale numeric formats (“1.234,56” vs “1,234.56”).

#### Node: Fraud Risk Check
- **Type / role:** `n8n-nodes-base.if` — conditional branching (fraud vs non-fraud).
- **Configuration (interpreted):**
  - The JSON does not show conditions. Common patterns:
    - `risk_score >= threshold`
    - `flags contains "bank_details_changed"`
    - `vendor not in whitelist`
- **Inputs / outputs:**
  - **Input:** from **Parse AI Response**
  - **Output (True branch / index 0):** to **Send Alert** and also to **Log to Financial Database**

- **Important behavior note:** In this workflow, **both branches log to Google Sheets**; the difference is that the “true” branch additionally alerts Slack.
- **Edge cases / failures:**
  - Condition referencing a missing field → IF evaluation may behave unexpectedly.
  - Wrong threshold can spam Slack or miss real risks.

#### Node: Send Alert

- **Inputs / outputs:**

## 4. Reproducing the Workflow from Scratch

1. **Create a new workflow** in n8n and name it:  
   **“Analyze invoices and predict cash flow with Gemini, Google Sheets, Gmail, and Slack”**

2. **Add entry node: Form Trigger**
   1. Add node: **Form Trigger** → rename to **Invoice Upload Form**
   2. Configure form fields (recommended minimum):
      - Vendor/Supplier (text)
      - Invoice number (text)
      - Invoice date (date)
      - Due date (date)
      - Total amount (number)
      - Currency (text or select)
      - Optional: attachment upload (PDF/image) if supported in your n8n version
   3. Save to generate the form URL (n8n will manage the webhook internally).

3. **Add “Process Invoice Data” (Code node)**
   1. Add node: **Code** → rename to **Process Invoice Data**
   2. Implement logic to:
      - Read form submission fields from the incoming item
      - Build a normalized JSON object, e.g.:
        - `vendor`, `invoiceNumber`, `invoiceDate`, `dueDate`, `amount`, `currency`
      - If attachments are present, extract metadata and/or prepare OCR text input (if you plan to add OCR later)
   3. Connect: **Invoice Upload Form → Process Invoice Data**

4. **Add Gemini model for invoice analysis**
   1. Add node: **Google Gemini Chat Model (LangChain)** → rename to **Google Gemini Chat Model**
   2. Configure credentials for Gemini (API key / auth as required by your node).
   3. Set model options (suggested):
      - Lower temperature for extraction consistency.
   4. Add node: **Chain LLM (LangChain)** → rename to **AI Invoice Analyzer**
   5. In **AI Invoice Analyzer**, configure:
      - Prompt/chain instructions to return **strict JSON** (strongly recommended), including fields like:
        - `vendor`, `invoice_number`, `invoice_date`, `due_date`, `total`, `currency`
        - `risk_score` (0–100), `risk_flags` (array), `risk_reason` (string)
   6. Connect:
      - **Process Invoice Data → AI Invoice Analyzer**
      - **Google Gemini Chat Model (ai_languageModel) → AI Invoice Analyzer**

5. **Add “Parse AI Response” (Code node)**
   1. Add node: **Code** → rename to **Parse AI Response**
   2. Implement parsing:
      - If model output is text, extract JSON portion
      - `JSON.parse(...)` with try/catch
      - Validate required keys and set defaults
   3. Connect: **AI Invoice Analyzer → Parse AI Response**

6. **Add fraud decision (IF node)**
   1. Add node: **IF** → rename to **Fraud Risk Check**
   2. Configure condition(s), for example:
      - Number condition: `{{$json.risk_score}}` **is greater than or equal** `70`
      - Or string/array checks for flags
   3. Connect: **Parse AI Response → Fraud Risk Check**

7. **Add Slack alert**
   1. Add node: **Slack** → rename to **Send Alert**
   2. Configure Slack credentials (OAuth2 or bot token depending on your n8n setup).
   3. Choose operation: **Post message**
   4. Set channel and message template (include vendor, amount, risk_reason, link to sheet row if available).
   5. Connect: **Fraud Risk Check (true output) → Send Alert**

8. **Add Google Sheets logging**
   1. Add node: **Google Sheets** → rename to **Log to Financial Database**
   2. Configure Google Sheets OAuth2 credentials.
   3. Select Spreadsheet + Sheet/Tab.
   4. Operation: typically **Append row**.
   5. Map columns from parsed fields (vendor, invoice number, totals, due date, risk score, etc.).
   6. Connect:
      - **Fraud Risk Check (true output) → Log to Financial Database**
      - **Fraud Risk Check (false output) → Log to Financial Database**

9. **Add Gemini model for cash-flow prediction**
   1. Add node: **Google Gemini Chat Model (LangChain)** → rename to **Google Gemini Chat Model1**
   2. Configure credentials (can reuse the same Gemini credential).
   3. Add node: **Chain LLM (LangChain)** → rename to **Cash Flow Predictor**
   4. Configure prompt/chain to output forecast fields, e.g.:
      - `expected_payment_date`
      - `cash_impact_week`
      - `notes` / `confidence`
   5. Connect:
      - **Log to Financial Database → Cash Flow Predictor**
      - **Google Gemini Chat Model1 (ai_languageModel) → Cash Flow Predictor**

10. **Add Gmail reporting**
   1. Add node: **Gmail** → rename to **Send Financial Report**
   2. Configure Gmail OAuth2 credentials (ensure send scope).
   3. Set:
      - Recipient(s) (finance mailbox)
      - Subject (include invoice number/vendor)
      - Body: include extracted invoice details + cash-flow prediction
   4. Connect: **Cash Flow Predictor → Send Financial Report**

11. **Activate workflow** and test end-to-end by submitting the form.
